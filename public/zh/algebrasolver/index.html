为什么存在这样的页面

算术错误很少是戏剧性的——它们是安静的。电子表格中的数字调换、符号在推导过程中翻转、舍入步骤太早应用了一行。代价是您通常不会注意到，直到错误的号码已被用于某些用途。通过独立实现运行相同的数字是可用的复式输入检查的最便宜的形式，这正是本页面的目的。

网络上有很多工具可以通过首先要求提供电子邮件地址、cookie 决策和四秒钟的广告来解决三十秒的问题。这笔交易对某些人来说是有意义的，但对只想要数字的人来说却没有意义。此页面采取相反的立场：没有注册，没有插页式广告，没有追加销售，一旦你得到了你想要的东西就没有理由离开。

结果告诉你什么，没有告诉你什么

这里的每个结果每次都是由相同的确定性例程产生的。没有采样，没有运行之间变化的近似阶段，也没有从之前的输入中继承的隐藏状态。明天给它相同的数字，您将得到相同的答案，当您将结果与同事或教科书进行核对时，这一点很重要。

明确界限是值得的，因为工具通常会受到超出其应得信任的信任。这个输入是诚实的：给它准确的值，它就会给你一个准确的计算。它无法告诉您所提供的值是否正确，您是否提出了正确的问题，或者您应用答案的上下文是否具有输入从未捕获的重要内容。这种判断由你来决定，这才是真正重要的部分。

值得避免的常见错误

第一个是信任您未检查其输入的结果。工具无法知道您打算输入一个数字并输入另一个数字，并且它会以与正确答案完全相同的置信度呈现错误答案。在依赖输出之前读回您的输入 - 只需两秒钟即可发现大部分问题。

第二种是将一个数字视为对从未被问过的问题的答案。本页上的每个计算都回答了一些狭隘而具体的问题。将其应用于附近但不同的问题是人们最终被一直运行良好的工具误导的最常见方式。如果您不确定您遇到的问题是否与工具回答的问题相符，那么这种不确定性就是有用的信号，而不是障碍。

关于准确性的说明

结果使用标准双精度算术进行计算，并显示到合理数量的位置，而不是硬件可以生成的每个数字。显示两位小数的数字并不声称精确到两位小数 - 它声称您提供的输入按面值计算得出该数字。显示精度和现实世界中的准确性是不同的事情，将它们混为一谈是正确工具产生错误决策的最常见方式。

如果结果看起来有误，请在怀疑算术之前检查输入。实际上，绝大多数令人惊讶的答案都来自于以错误的单位输入的值、小数分隔符的读数与您预期的不同，或者应该更改的字段保留为默认值。这并不是对任何人的批评——这只是错误所在，知道先看哪里可以节省最多时间。

代数求解器常见问题解答

线性方程如何求解？
ax + b = c → x = (c - b) / a。示例：2x + 3 = 7 → x = (7-3)/2 = 2。
方程组是如何求解的？
使用带有 2×2 行列式的克莱默法则。对于 a1x + b1y = c1 和 a2x + b2y = c2：D = a1b2 - a2b1。 x = (c₁b2 - c2b₁) / D。 y = (a₁c2 - a2c₁) / D。
如果没有解决办法怎么办？
当 D = 0 时，系统没有唯一解。如果 D = 0 并且分子也为 0，则系统有无穷多个解（相关方程）。如果 D = 0 但分子不为 0，则无解（方程不一致）。
